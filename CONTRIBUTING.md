# Contributing to HomeLab Dashboard 🚀

First off, thank you for your interest in contributing to HomeLab Dashboard! Even though this is a solo-developed project, community feedback is what makes it better.

## 🛑 Important Notice: Code Contributions (Pull Requests)
At this stage of development (Beta), **I am currently NOT accepting direct code contributions (Pull Requests)**. 

Maintaining a strict quality control and a specific architectural vision for this project is my priority right now. If you submit a PR with code changes, it will politely be closed. Thank you for understanding!

However, there are many other, equally important ways you can contribute to the project:

## 🐛 1. Reporting Bugs
Found a bug? The Docker scanner crashed? A widget is not updating? Please let me know!
* Go to the **Issues** tab.
* Select the **Bug Report** template.
* Fill in as much detail as possible (Docker version, browser, steps to reproduce, and screenshots).

## 💡 2. Suggesting Features & New Widgets
Have a great idea for a new feature or a specific service widget?
* Go to the **Issues** tab.
* Select either the **Feature Request** or **New Widget Request** template.
* Explain why this feature would be useful and provide links to the official documentation/API of the service if you are requesting a widget.

## 🧪 3. Testing the Beta
Since this is an Early Preview (v0.1.0-beta), the best way to help is to simply use the dashboard in your own HomeLab setup. Test different containers, try to break the grid layout, and report your findings using the Issues tab.

## 🌟 4. Spread the Word & Support
If you like the dashboard, the easiest way to contribute is to:
* Give this repository a ⭐️ on GitHub.
* Share it on Reddit (r/selfhosted) or your favorite HomeLab community.
* Consider supporting the late-night coding sessions through [GitHub Sponsors](https://github.com/sponsors/Vesse00).

---
*By participating in this project, you agree to maintain a respectful and polite environment in all discussions and issues.*
